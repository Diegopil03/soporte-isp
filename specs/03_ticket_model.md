# Modelo de ticket

## Objetivo
Definir los campos mínimos que componen un ticket operativo.

## Campos obligatorios
- ID interno
- ID WispHub
- Fecha de creación
- Nombre del cliente
- Zona / fraccionamiento
- Tipo de incidencia
- Estado
- Prioridad
- Bloque de tiempo
- Tipo de cliente
- Técnico asignado

## Campos opcionales
- Teléfono
- Referencias
- Evidencia
- Observaciones

## Campos internos
- Prioridad sugerida
- Afectación múltiple
- Antigüedad

## Campos automáticos
- Tiempo abierto
- Historial de cambios
